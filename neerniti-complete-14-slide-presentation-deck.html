Technical Details

Embed model: ibm/slate-125m-english-rtrvr via watsonx.ai

Vector DB: FAISS IndexFlatIP (768-dim, cosine similarity)

Chunk store: 36 state JSON chunks saved in IBM COS

LLM: ibm/granite-3-2-8b-instruct · temp 0.2 · max_tokens 300

Latency: <3 seconds end-to-end on Cloud Functions

Why RAG over Fine-tuning?

MIS-78 data is factual & structured — retrieval beats memorization

Updatable — swap CSV, rebuild FAISS index, same model

Verifiable — every answer cites the retrieved state chunk

Free tier — FAISS runs in-memory, no vector DB subscription needed
